# AI Change Review Checklist

- [ ] Change scope is clear.
- [ ] Diff is small enough to review.
- [ ] Tests match the risk.
- [ ] Failing or skipped tests are explained.
- [ ] Observability impact is considered.
- [ ] Security and dependency impact is considered.
- [ ] Generated code is reviewed like human code.
- [ ] What remains unproven is documented.
- [ ] Recommendation is approve, revise, or reject.
