# Workshop 4 Expanded - Validation: Tests, Observability, Security

This expanded version turns Workshop 4 into a validation gate for AI-assisted changes.

## Thesis

AI-generated change plus validation equals trustworthy improvement.

Participants take a candidate change from Workshop 3 and test whether it is safe enough to merge, revise, or discard.

## Recommended Delivery

- Day 1: build a validation matrix for the candidate branch.
- Day 2: evaluate testing confidence and add or propose lockdown tests.
- Day 3: review observability and operational evidence.
- Day 4: review security, secrets, dependencies, and supply-chain risk.
- Day 5: run a validation gate and prepare group approval/rescore.
