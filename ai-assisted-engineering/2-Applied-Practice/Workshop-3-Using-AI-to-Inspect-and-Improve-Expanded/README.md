# Workshop 3 Expanded - Using AI To Inspect And Improve

This expanded version turns Workshop 3 into a hands-on workflow for using Claude Code to inspect a real codebase, tighten prompts, plan a small improvement, make a candidate change, and review the result.

## Thesis

AI is most useful on existing codebases when it is aimed at a specific question, constrained by evidence, and forced through human review.

## Recommended Delivery

- Day 1: build the inspection plan from Workshop 2 friction priorities.
- Day 2: run a codebase understanding pass and capture evidence.
- Day 3: tighten prompts and separate evidence from inference.
- Day 4: create a small change plan and apply one candidate improvement.
- Day 5: review the diff, validate narrowly, and prepare group handoff.
