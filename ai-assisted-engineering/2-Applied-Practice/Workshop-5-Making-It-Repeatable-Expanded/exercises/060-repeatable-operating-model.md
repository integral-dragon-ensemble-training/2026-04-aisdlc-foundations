# 060 Repeatable Operating Model

## Objective

Define the team's repeatable AI-SDLC operating model.

## Claude Prompt

```text
Create a repeatable AI-SDLC operating model for this team.

Include:
- how the team identifies friction
- how work is selected
- how Claude Code is used
- branch conventions
- evidence artifacts
- validation gates
- group review cadence
- merge/backlog decision rules
- metrics to revisit monthly

Keep it practical and short enough to adopt.
```

## Output

- repeatable operating model
- monthly review checklist
