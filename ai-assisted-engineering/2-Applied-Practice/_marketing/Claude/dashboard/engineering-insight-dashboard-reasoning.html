ADO Estimate-variance on Claims Adjudication area path, last 90 days

Pull work items closed in the last 90 days on the target area path, then compare Original Estimate against Completed Work. Roll up the variance by epic.

# list completed work items on the target area path in last 90 days
az boards query --org https://dev.azure.com/ensemble --project ClaimsAdjudication \
  --wiql "SELECT [System.Id], [System.Title],
              [Microsoft.VSTS.Scheduling.OriginalEstimate],
              [Microsoft.VSTS.Scheduling.CompletedWork]
       FROM WorkItems
       WHERE [System.AreaPath] UNDER 'ClaimsAdjudication\\Submission'
         AND [System.State] = 'Closed'
         AND [System.ChangedDate] >= @today-90" \
  --output json | jq '[ .[] | {id, est: .["Microsoft.VSTS.Scheduling.OriginalEstimate"],
                  act: .["Microsoft.VSTS.Scheduling.CompletedWork"]} ]'
GIT Files that repeatedly co-change on over-estimate PRs

For each work item with variance > 50 %, fetch its linked PRs and produce a co-change matrix for the files touched.

# files + commit count in the 90-day window, sorted by churn
git log --since=3.months --numstat --pretty= -- src/submission \
  | awk '{print $3}' | sort | uniq -c | sort -rn | head -20

# co-change frequency: pairs of files that changed together in the same commit
git log --since=3.months --name-only --pretty=format:"---%h" \
  | awk '/^---/{ if (n>1) for (i in f) for (j in f) if (i<j) print f[i], f[j]; n=0; delete f; next } { f[++n]=$0 }' \
  | sort | uniq -c | sort -rn | head -20

  | awk '
      /^---/ {
          sub=""; pay=""; ctrl="";
          for (i in f) {
              if (f[i] ~ /SubmissionService/) sub=1
              if (f[i] ~ /PayerMapper/)      pay=1
              if (f[i] ~ /controllers/)      ctrl=1
          }
          if (sub+pay+ctrl >= 2) print h
          h=$0; delete f; next
      }
      { f[++n]=$0 }'
