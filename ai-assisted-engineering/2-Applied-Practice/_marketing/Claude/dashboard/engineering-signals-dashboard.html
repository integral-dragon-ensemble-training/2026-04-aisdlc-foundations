What this dashboard intentionally does not do
No cost conversions.
We did not multiply cycle time by engineer rate to produce a dollar number. Those numbers are defensible only when an executive accepts a range, and they are not 95%-confidence outputs. If you want them, request them — we'll show them as "32–50 engineer-hours" not "42."
No qualitative friction narrative.
"Claims submission validation is entangled with persistence" requires reading the code and making a judgment. That judgment is valuable — it just isn't in this dashboard. It belongs in the memo that accompanies it.
No per-engineer attribution.
Author names appear in aggregate (unique-author count) and nowhere else. This dashboard is about the system, not the people.
No forward-looking forecasts.
"Payer integration drift" belongs in the narrative memo, not here. This page reports what happened, not what's about to.
