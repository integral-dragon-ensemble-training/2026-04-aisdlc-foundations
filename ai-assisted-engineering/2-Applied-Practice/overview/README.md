# Applied Practice — Week 3

Welcome to the Applied Practice week of the AI-Accelerated Software Development program. This is the second of three weeks, building on Foundations and preparing you for the Applied week that follows.

## What this week is about

Over the next five sessions, you'll work through a practical question:

> **Where is change in your software hardest, and how can AI-assisted engineering make it easier?**

Each day builds on the last. You'll start by defining what a healthy software project looks like, then learn to recognize the friction that slows teams down, then practice using AI to inspect and improve real code — and finish with a concrete plan you can take back to your own team.

## What you'll take away

By the end of the week, you'll be able to:

- assess the health of a software project using a structured rubric,
- identify and prioritize the technical debt that most blocks change,
- use AI tools to inspect, explain, and improve an existing codebase,
- validate AI-assisted changes with tests, observability, and security checks,
- build a staged improvement plan your team can actually execute.

## How the week is structured

Five sessions, one per day, sixty minutes each. Each session mixes short concept framing, examples drawn from real codebases, and hands-on exercises you'll do individually or in small groups. You'll leave each day with an artifact — a scorecard, a friction map, a prompt, a validation checklist, or an improvement plan — that you can reuse on your own projects.
