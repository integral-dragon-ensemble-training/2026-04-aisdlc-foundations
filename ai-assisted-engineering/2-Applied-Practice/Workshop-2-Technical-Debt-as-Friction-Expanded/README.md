# Workshop 2 Expanded - Technical Debt As Friction

This expanded version preserves the original Workshop 2 and turns it into a hands-on Claude Code workflow for finding, classifying, and prioritizing friction in a real project.

## Thesis

Technical debt is not just messy code. It is the repeated tax the system charges on every change.

Participants use Claude Code to collect evidence, map friction, estimate interest, and choose a small debt-reduction candidate on their participant branch.

## Recommended Delivery

- Day 1: carry forward Workshop 1 scorecard findings and create the Workshop 2 branch/workspace.
- Day 2: run a friction inventory across docs, architecture, setup, tests, release, data, observability, security, and workflow.
- Day 3: classify debt by category and estimate interest rate.
- Day 4: prioritize by flow impact and select one candidate improvement.
- Day 5: compare participant friction maps and decide what belongs in the team backlog.
