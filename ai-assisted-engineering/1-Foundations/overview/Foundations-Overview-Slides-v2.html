Foundations  |  Why We Redesigned This
The shift: from vibe coding to structured agentic engineering
Vibe Coding  (Legacy approach) Agentic Engineering  ( This program's target)
Interface —Conversational prompting via external chatbots. Heavy copy-paste, no codebase awareness. ✓CLI-native orchestration. Agent reads the codebase, executes shell commands, and loops on failure.
Workflow Trigger —High-level, unconstrained prompts driving a "creative flow." Ideas prioritized over structure. ✓Spec-Driven Development: intent → constraints → acceptance criteria. Small, verifiable prompts.
Constraint Mechanism —Model generates without limits. Arbitrary network calls, unchecked mutations, architectural drift. ✓Explicit functional boundaries. CLAUDE.md rules, Plan Mode gating, and hook enforcement.
Quality & Review —Minimal oversight. "Trust debt" accumulates as engineers can't reconstruct why code exists. ✓Structured review discipline. AI output treated as untrusted until verified by tests or human review.
Security Posture —Brittle, opaque codebases. High risk of OWASP vulnerabilities, no audit trail for agent actions. ✓Zero Trust execution. Least-privilege permissions, immutable audit logs, OWASP-aware threat model.
Enterprise environments cannot tolerate vibe coding at scale. This curriculum instills the discipline that makes AI-accelerated delivery safe to ship.

Module 1  |  Sessions 1–4  ·  Week 1
The Agentic Paradigm and CLI Environment

Session 1: The Agentic Engineering Shift

Replace casual AI usage with a structured model: output, reviewability, and ownership all matter simultaneously. Comprehension debt is introduced as the first enterprise risk.

Session 2: LLM Internals for Orchestrators

Build mechanical sympathy — tokens, inference, variance, context windows — to make better workflow decisions. Only the theory that directly impacts engineering choices.

Session 3: Context Boundaries and Rot

Understand why context quality matters more than context volume. The minimum working set principle, the N-tool problem, and how overloaded agents lose focus and coherence.

Session 4: Claude Code CLI Architecture and Setup

Install and correctly bound the harness — context init, tool injection, trust boundary, and approval mode. The CLI becomes the primary engineering environment, not a supplementary tool.

Core message: AI writes syntax; engineers still own architecture, judgment, and liability.
Foundations Overview v2.0  |  Integral Dragon
Module 2  |  Sessions 5–8  ·  Week 1
Spec-Driven Development and Context Control

Session 5: Spec-Driven Development

Define machine-readable intent — goal, constraints, acceptance criteria, non-goals — before generation begins. Code without a preceding spec is a policy violation, not a shortcut.

Session 6: Plan Mode vs. Act Mode

Separate codebase analysis from mutation. Generate a plan in read-only mode, reject it, refine it, then approve before any file is touched. Prevents rogue agents from rewriting architecture.

Session 7: Context Control via CLAUDE.md

Persist project rules so constraints survive across sessions, engineers, and agents without re-prompting. Standardizes AI behavior across teams. Scoped: user, project, local, and managed.

Session 8: Extending Capabilities with MCP

Connect external systems through governed tool boundaries — client/server architecture, payload inspection, authorization, and blast-radius control. MCP is the 2026 enterprise standard.

Core message: Reliable agentic engineering starts with explicit intent and governed context.

Module 3  |  Sessions 9–12  ·  Week 2
Tooling, MCP, and Agentic Orchestration

Session 9: Skills and Deterministic Hooks

Decide what remains flexible (skills that scale team expertise) versus what becomes automatic enforcement (hooks that fire deterministically before every commit or file write).

Session 10: Task Decomposition and Subagents

Break large work into bounded delegated units. Give each subagent a clear contract, scope limit, and output interface. Prevent context collapse through atomic task design.

Session 11: Parallel Orchestration: Agent Teams

Coordinate multiple agents simultaneously without losing state discipline, collision control, or merge coherence. The N×M orchestration problem becomes manageable with explicit boundaries.

Session 12: OWASP Top 10 for Agentic Apps

Secure the broader execution surface created by tooling, teams, and expanded capability. Prompt injection, indirect tool calls, unauthorized escalation, and MCP server compromise.

Core message: Once the workflow is constrained, capability can expand safely.

Module 4  |  Sessions 13–16  ·  Week 2
Enterprise Security, Governance, and Auditability

Session 13: Zero Trust and Permission Boundaries

Apply least privilege to agent execution — trust no implicit scope, verify all access. Configure role-based boundaries so subagents cannot escalate beyond their defined mission.

Session 14: Secure MCP Governance

Govern external tool and MCP trust boundaries — authorization tokens, payload inspection, blast radius containment, and server compromise detection.

Session 15: Code Review and Cognitive Debt Mitigation

Review AI-generated diffs for structural risk and comprehension debt, not just surface correctness. A passing test is not a substitute for a reviewable change.

Session 16: Auditability and Workflow Narration

Preserve enough evidence to reconstruct why a change happened — narration discipline, commit traceability, and the compliance record that enterprise delivery requires.

Core message: Enterprise use is not real until permissions, review, and traceability hold under scrutiny.

Foundations  |  Outcomes
What learners can do by the end of Foundations
✓Explain the difference between vibe coding and structured agentic engineering — and why it matters for enterprise delivery
✓Choose the smallest useful context for a task and defend that choice to a reviewer
✓Drive work from specs and reviewed plans rather than open-ended prompts
✓Operate Claude Code inside explicit permission and context boundaries configured by the team
✓Decompose work across tools and agents with clear delegation contracts and scope limits
✓Review and narrate AI-assisted changes in terms a team can inherit, audit, and trace back to intent
