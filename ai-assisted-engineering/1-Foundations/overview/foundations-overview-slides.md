# Foundations Overview
## Modules 1 through 4
### Agentic engineering foundations before labs begin

---

# What Foundations is for

- replace ad hoc AI usage with a disciplined engineering operating model
- establish the CLI-native working environment
- teach control surfaces before acceleration features
- prepare learners to enter labs with repeatable habits

---

# The four-module arc

- Module 1: adopt the agentic engineering posture and the CLI environment
- Module 2: constrain intent, execution, and context
- Module 3: extend capability and orchestrate delegated work
- Module 4: secure execution, review, and auditability

---

# Module 1
## The Agentic Paradigm and CLI Environment

- Session 1: The Agentic Engineering Shift
- Session 2: LLM Internals for Orchestrators
- Session 3: Context Boundaries and Rot
- Session 4: Claude Code CLI Architecture and Setup

Core message:
- AI writes syntax; engineers still own architecture, judgment, and liability.

---

# Module 1 teaching promise

- stop treating AI as a chat trick
- understand token limits, variance, and context pressure
- learn why bounded context matters more than raw context volume
- install and verify the Claude Code harness in a controlled way

---

# Module 2
## Spec-Driven Development and Context Control

- Session 5: Spec-Driven Development
- Session 6: Plan Mode vs. Act Mode
- Session 7: Context Control via CLAUDE.md
- Session 8: Extending Capabilities with MCP

Core message:
- reliable agentic engineering starts with explicit intent and governed context.

---

# Module 2 teaching promise

- define machine-readable intent before generation
- separate planning from mutation
- persist project rules so constraints survive across sessions
- connect external systems through governed capability boundaries

---

# Module 3
## Tooling, MCP, and Agentic Orchestration

- Session 9: Skills and Deterministic Hooks
- Session 10: Task Decomposition and Subagents
- Session 11: Parallel Orchestration: Agent Teams
- Session 12: OWASP Top 10 for Agentic Apps

Core message:
- once the workflow is constrained, capability can expand safely.

---

# Module 3 teaching promise

- decide what remains flexible versus what becomes deterministic
- break large work into bounded delegated units
- coordinate multiple agents without losing state discipline
- secure the broader execution surface created by tooling and teams

---

# Module 4
## Enterprise Security, Governance, and Auditability

- Session 13: Zero Trust and Permission Boundaries
- Session 14: Secure MCP Governance
- Session 15: Code Review and Cognitive Debt Mitigation
- Session 16: Auditability and Workflow Narration

Core message:
- enterprise use is not real until permissions, review, and traceability hold under scrutiny.

---

# Module 4 teaching promise

- apply least privilege to agent execution
- govern external tool and MCP trust boundaries
- review AI-generated diffs for structural risk, not just surface correctness
- preserve enough evidence to reconstruct why a change happened later

---

# The control stack learners build

1. posture and mental model
2. bounded context and explicit intent
3. deterministic enforcement and decomposition
4. permissions, review discipline, and auditable evidence

This is the path from AI assistance to enterprise-grade agentic engineering.

---

# Session progression at a glance

- Sessions 1 to 4: posture, mechanics, context, harness
- Sessions 5 to 8: specs, plans, persistent rules, governed tools
- Sessions 9 to 12: hooks, subagents, teams, security threats
- Sessions 13 to 16: permissions, governance, review, traceability

Each module adds control before adding more power.

---

# What learners should be able to do by the end of Foundations

- explain the difference between vibe coding and agentic engineering
- choose the smallest useful context for a task
- drive work from specs and reviewed plans
- operate Claude Code inside explicit boundaries
- decompose work across tools and agents safely
- review and narrate AI-assisted changes in enterprise terms

---

# Bridge to Labs

- labs assume the learner can already work in the CLI
- labs assume bounded scope and review discipline are normal
- labs assume AI output is untrusted until verified
- labs build on this foundation with codebase reconnaissance, testing, and CI loops

---

# Key takeaway

- Foundations is not a theory block.
- It is the operating baseline for trustworthy AI-accelerated delivery.
- The goal is not faster prompts.
- The goal is controlled, reviewable, enterprise-ready engineering work.
