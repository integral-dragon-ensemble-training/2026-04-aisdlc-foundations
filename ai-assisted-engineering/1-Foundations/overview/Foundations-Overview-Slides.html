Foundations  |  Purpose
Foundations is not a theory block — it is the operating baseline
→Replace ad hoc AI usage with a disciplined engineering operating model
→Establish the CLI-native working environment before acceleration features are added
→Teach control surfaces — specs, plans, rules, permissions, review — before scope expands
→Prepare learners to enter Labs with repeatable habits, not one-off prompt tricks
The goal is not faster prompts.
The goal is controlled, reviewable, enterprise-ready engineering work.

Module 1  |  Sessions 1–4
The Agentic Paradigm and CLI Environment

Session 1: The Agentic Engineering Shift

Replace casual AI usage with a structured model: output, reviewability, and ownership all matter simultaneously.

Session 2: LLM Internals for Orchestrators

Build mechanical sympathy — tokens, inference, variance, context windows — to make better workflow decisions.

Session 3: Context Boundaries and Rot

Understand why context quality matters more than context volume, and how to apply the minimum working set principle.

Session 4: Claude Code CLI Architecture and Setup

Install and correctly bound the harness — context init, tool injection, trust boundary, approval mode.

Core message: AI writes syntax; engineers still own architecture, judgment, and liability.
Foundations Overview  |  Integral Dragon
Module 2  |  Sessions 5–8
Spec-Driven Development and Context Control

Session 5: Spec-Driven Development

Define machine-readable intent — goal, constraints, acceptance criteria, non-goals — before generation begins.

Session 6: Plan Mode vs. Act Mode

Separate codebase analysis from mutation. Review the plan before approving execution.

Session 7: Context Control via CLAUDE.md

Persist project rules so constraints survive across sessions, engineers, and agents without re-prompting.

Session 8: Extending Capabilities with MCP

Connect external systems through governed tool boundaries — security, payload legibility, and authorization.

Core message: Reliable agentic engineering starts with explicit intent and governed context.

Module 3  |  Sessions 9–12
Tooling, MCP, and Agentic Orchestration

Session 9: Skills and Deterministic Hooks

Decide what remains flexible (skills) versus what becomes automatic enforcement (hooks).

Session 10: Task Decomposition and Subagents

Break large work into bounded delegated units. Give each subagent a clear contract and scope.

Session 11: Parallel Orchestration: Agent Teams

Coordinate multiple agents without losing state discipline or collision control.

Session 12: OWASP Top 10 for Agentic Apps

Secure the broader execution surface created by tooling, teams, and expanded capability.

Core message: Once the workflow is constrained, capability can expand safely.

Module 4  |  Sessions 13–16
Enterprise Security, Governance, and Auditability

Session 13: Zero Trust and Permission Boundaries

Apply least privilege to agent execution — trust no implicit scope, verify all access.

Session 14: Secure MCP Governance

Govern external tool and MCP trust boundaries — authorization, payload inspection, blast radius.

Session 15: Code Review and Cognitive Debt Mitigation

Review AI-generated diffs for structural risk and comprehension debt, not just surface correctness.

Session 16: Auditability and Workflow Narration

Preserve enough evidence to reconstruct why a change happened — narration, traceability, compliance.

Core message: Enterprise use is not real until permissions, review, and traceability hold under scrutiny.

Foundations  |  Session Map
Session progression at a glance
S01–S04
Posture, mechanics, context quality, and CLI harness setup. The mental model and the environment before any tooling is added.
S05–S08
Specs, plans, persistent project rules, and governed external tools. Intent becomes explicit; context becomes bounded.
S09–S12
Hooks, subagents, parallel agent teams, and the OWASP threat surface. Capability expands inside the constraints already in place.
S13–S16
Zero trust permissions, MCP governance, cognitive debt review, and auditable narration. Enterprise-grade delivery requires all of these simultaneously.
→Each module adds control before adding more power. The sequence is intentional.

Foundations  |  Outcomes
What learners can do by the end of Foundations
✓Explain the difference between vibe coding and structured agentic engineering
✓Choose the smallest useful context for a task and defend that choice
✓Drive work from specs and reviewed plans rather than open-ended prompts
✓Operate Claude Code inside explicit permission and context boundaries
✓Decompose work across tools and agents with clear delegation contracts
✓Review and narrate AI-assisted changes in terms a team can inherit and audit
Labs assume these capabilities are already in place. Learners who enter Labs without this foundation will be working at a disadvantage from the first exercise.

Labs assume Foundations is already in practice — not just understood
