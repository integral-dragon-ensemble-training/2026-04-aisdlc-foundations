# Module 2 Review and Module 3 Preview

## Review Scope

This note reviews the current authored state of Module 2 and previews the instructional arc for Module 3.

## Module 2 Review

### Current session sequence

Module 2 now forms a coherent four-session progression:

1. `Session 05` establishes `Spec-Driven Development` as the control point for intent.
2. `Session 06` adds `Plan Mode vs. Act Mode` as the control point for safe execution.
3. `Session 07` adds `CLAUDE.md` as the control point for durable project rules.
4. `Session 08` adds `MCP` as the control point for external capability extension.

### What is working well

- The module has a clear logic chain:
  - define intent
  - review execution
  - persist constraints
  - extend capability
- Each session is framed in enterprise terms rather than consumer AI terms.
- The module consistently reinforces reviewability, bounded scope, and governance.
- Session 07 and Session 08 create a strong bridge from internal harness controls to external system integration.

### Recommended delivery framing

Present Module 2 as the point where the cohort stops treating the model as a chat partner and starts treating the workflow as an engineered system.

That message is already present in the authored material and should be made explicit in live delivery.

### Minor gaps to close before live delivery

- Package Session 07 and Session 08 artifacts so Module 2 is fully presentation-ready.
- Reuse one recurring example across Sessions 05 through 08 if possible.
  - Suggested example themes:
    - authentication change
    - billing workflow
    - regulated API update
- Add one explicit bridge sentence in live facilitation:
  - Module 2 teaches how to constrain and connect the agent.
  - Module 3 teaches how to extend, decompose, and orchestrate the work.

### Module 2 teaching promise

By the end of Module 2, learners should understand that reliable agentic engineering depends on four things:

- a clear spec
- a reviewed plan
- durable project rules
- governed tool access

## Module 3 Preview

### Module label

`Module 3: Tooling, MCP, and Agentic Orchestration`

### Instructional promise

Module 3 should feel like the operational expansion of Module 2.

If Module 2 answered:

- What do we want?
- How do we constrain it?
- How do we review it?
- How do we connect it?

Then Module 3 answers:

- How do we extend capability safely?
- How do we automate enforcement?
- How do we break large work into isolated units?
- How do we coordinate multiple agents without losing control?

### Session sequence

1. `Session 09: Skills & Deterministic Hooks`
   - distinguish flexible reusable workflows from rigid enforcement gates
2. `Session 10: Task Decomposition & Subagents`
   - teach atomic task slicing and isolated delegated work
3. `Session 11: Parallel Orchestration: Agent Teams`
   - teach concurrent coordination and shared-state risks
4. `Session 12: OWASP Top 10 for Agentic Apps`
   - secure the expanded execution surface

### Recommended teaching emphasis

- Session 09 should feel like the answer to:
  - What should remain probabilistic, and what must become deterministic?
- Session 10 should feel like the answer to:
  - How do we keep complexity from collapsing the context window?
- Session 11 should feel like the answer to:
  - What changes when multiple workers operate at once?
- Session 12 should feel like the answer to:
  - What new attack and governance surface did we just create?

### Delivery note

Module 3 should not be presented as "more powerful features."

It should be presented as the next layer of operating discipline after Module 2. Otherwise the cohort may hear subagents, hooks, and teams as acceleration features rather than control features.
