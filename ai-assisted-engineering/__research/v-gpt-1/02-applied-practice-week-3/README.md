# AI-SDLC Applied Practice Course Bundle

Version: 0.1.0

This bundle is organized so you can teach the material immediately, then still reuse the same structure for later expansion into a longer course.

## Core thesis

**AI-assisted engineering is most valuable when it helps teams see, reduce, and systematically pay down the friction and technical debt that make software hard to change.**

## Core teaching approach

1. Define what “good” looks like.
2. Identify the debt that blocks it.
3. Use AI to analyze and improve it.
4. Validate improvements with tests, observability, and security.
5. Turn that into a repeatable engineering practice.

## Recommended use

- Start with `01-course-foundation/`.
- Build or adapt your class narrative from `02-course-design/`.
- Pull slide copy from `03-slides/`.
- Run the activities from `04-exercises/`.
- Use `05-resources/` for readings, public repos, and your Gemini Deep Research prompt.

## Design choice

The course is intentionally **not** organized as “AI writes code faster.”

It is organized as:

- good engineering fundamentals make software easier to change,
- technical debt increases the cost of every future change,
- AI tools amplify teams that can already see quality clearly,
- AI can help teams inspect, document, test, refactor, and standardize,
- but AI without validation can easily create faster messes.

## Teaching principle

Treat AI as a **force multiplier for engineering judgment**, not as a substitute for it.
