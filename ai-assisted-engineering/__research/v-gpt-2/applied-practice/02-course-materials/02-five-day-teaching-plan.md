# Five-Day Teaching Plan

## Day 1 — Define “what good looks like”
### Objective
Give students a practical model of software quality as changeability.

### Lessons
1. Changeability as the core quality target
2. Repo health scorecard and technical debt prioritization

### Main demo
Use Claude Code CLI to inspect a repo and produce a first-pass engineering health summary.

### Main student artifact
Completed scorecard and top-three blocker list.

---

## Day 2 — Make the system understandable and reproducible
### Objective
Teach students to improve architecture clarity and local setup reliability.

### Lessons
1. Architecture reconnaissance from code
2. README vs reality and minimum viable local run path

### Main demo
Claude Code CLI generates a first-pass architecture map and setup gap report.

### Main student artifact
Architecture notes plus smallest high-value setup improvement.

---

## Day 3 — Make changes verifiable
### Objective
Teach students to strengthen confidence through testability, tests, and data discipline.

### Lessons
1. Testability before test count
2. Data, migrations, and repeatable verification

### Main demo
Claude Code CLI diagnoses testability issues and proposes small seam-creating changes.

### Main student artifact
Test improvement sketch and migration/test-data risk list.

---

## Day 4 — Make the running system visible
### Objective
Teach students to improve observability and release verification.

### Lessons
1. Logs, metrics, traces, and health
2. Version visibility and deploy verification

### Main demo
Claude Code CLI finds observability and release verification gaps.

### Main student artifact
One small observability improvement proposal plus a deploy-verification checklist.

---

## Day 5 — Turn improvement into a repeatable practice
### Objective
Teach students to prioritize and sequence debt reduction safely.

### Lessons
1. Guardrails for AI-assisted change
2. Capstone planning and operating model

### Main demo
Claude Code CLI drafts a two-week incremental debt-paydown plan.

### Main student artifact
Capstone plan with scorecard, blockers, improvement sequence, and review gates.
