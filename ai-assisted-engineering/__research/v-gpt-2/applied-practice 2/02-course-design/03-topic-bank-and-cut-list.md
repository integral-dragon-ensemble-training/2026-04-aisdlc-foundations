# Topic Bank and Cut List

Use this file when you need to adjust the course for time.

## Must-teach topics

- the main thesis of AI-assisted engineering,
- what good looks like,
- technical debt as friction,
- targeted AI use cases,
- validation via tests, observability, and security,
- staged improvement instead of rewrite thinking.

## Strongly recommended topics

- architecture diagrams and boundary clarity,
- reproducible developer setup,
- test strategy versus vanity coverage,
- release discipline and version visibility,
- repository health scorecard.

## Nice-to-have topics

- semantic versioning automation,
- conventional commits,
- advanced supply-chain assurance concepts,
- deeper incident analysis examples,
- multiple public repo walkthroughs.

## Topics to cut first if time gets tight

1. deep tool-specific workflow configuration,
2. advanced release automation detail,
3. long discussion of branching strategy,
4. fine-grained testing philosophy debates,
5. too many public repo tours.

## Topics to expand if you later turn this into a longer course

- architecture refactoring patterns,
- contract testing,
- migration patterns and data safety,
- structured observability design,
- AI workflow governance and policy,
- scorecards and quality dashboards,
- real codebase clinics using student repos.
