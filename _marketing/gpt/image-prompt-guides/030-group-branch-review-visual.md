# Prompt Guide: Group Branch Review Visual

## Use Case

Use this to explain why participant branches and group review reduce risk while improving learning and adoption.

## Base Prompt

```text
Draw a 16:9 executive presentation visual showing an AI-assisted team workflow for safe experimentation and group adoption.

Style: modern enterprise workflow diagram, crisp vector illustration, light background, slate blue, teal, and warm gray palette, clear spacing, professional and calm.

Show three engineers working in parallel on separate safe branches from the same real project repository. Each branch has an AI assistant helping inspect, document, or test the code. The branches flow into a central Group Review checkpoint. From Group Review, show two paths:
Adopt
Backlog

Add a small evidence tray near the review checkpoint with these labels:
Diff Summary
Validation Evidence
Insight Log

Make clear that nothing merges directly to production. Show production as a protected area after the Group Review checkpoint.

Avoid robots taking control, fake names, fake logos, busy UI screens, and dense text. Make labels large enough for a leadership slide.
```

## Variant: More Governance Emphasis

```text
Revise the visual to emphasize governance: add a protected gate between Group Review and Production with the short label "Validation Gate." Keep the image uncluttered and do not add more than six labels total.
```

## Variant: More Team Learning Emphasis

```text
Revise the visual to emphasize team learning: show the three branches producing shared lessons and selected team standards. Keep the same safe branch and group review structure.
```

## Suggested Slide Title

```text
Parallel AI Experiments, Shared Review, Controlled Adoption
```
