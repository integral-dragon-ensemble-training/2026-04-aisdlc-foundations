# Prompt Guide: Before And After Insights Visual

## Use Case

Use this to sell the "insights, not metrics" framing. The visual should make it clear that the program turns hidden engineering friction into visible decisions.

## Base Prompt

```text
Draw a 16:9 executive presentation visual comparing "Before" and "After" for an AI-assisted engineering program.

Style: premium enterprise strategy deck, clean infographic, off-white background, charcoal text, restrained blue and amber accents, simple geometric cards, no clutter.

Left side title: Before
Show a foggy codebase silhouette with three short labels:
Hidden Friction
Unclear Risk
Local AI Usage

Right side title: After
Show a clear executive insights board with five large readable cards:
Architecture Clarity
Setup Friction
Testing Confidence
Validation Evidence
Adoption Decisions

Between the two sides, show a simple transformation arrow labeled:
Applied Practice

Make the right side feel more structured and actionable, but do not show numeric metrics, surveillance, individual rankings, or fake tiny charts. Keep all text large and readable.
```

## Variant: More Visual, Less Text

```text
Regenerate this as a mostly visual metaphor with fewer labels: a foggy tangled system on the left becoming a clear map and decision board on the right. Use only these labels: Before, Applied Practice, After, Insights, Decisions.
```

## Variant: Dashboard Mockup

```text
Create a realistic but abstract executive dashboard mockup for the "After" side. Use large readable category labels only. Do not include fake numbers, detailed charts, or tiny text.
```

## Suggested Slide Title

```text
Applied Practice Turns Hidden Engineering Friction Into Actionable Insights
```
